## 内存

Go语言中，常量在**编译期**就被分配内存。这意味着常量的值在程序运行之前就已经确定，并且会直接嵌入到生成的可执行文件中。

具体来说，Go编译器会在编译阶段解析和计算常量表达式，并将常量值直接存储到程序的数据段中。数据段是可执行文件中存储全局变量和常量的内存区域。

由于常量的值是固定的，编译器可以直接将其替换为字面值。因此，在运行时并不需要为常量分配内存空间，也不需要为其保存内存地址。

如下代码：

```go
const PI = 3.1415926535
```

编译该代码时，Go编译器会将 `PI` 的值直接嵌入到生成的可执行文件中；在程序运行时，`PI` 的值可以直接从数据段中读取，而无需进行任何内存分配操作。

**与变量不同，常量的值在运行时是不可修改的。** 因此，Go语言不会为常量分配堆内存空间。**内存四区**的概念可以参看这篇文章。
